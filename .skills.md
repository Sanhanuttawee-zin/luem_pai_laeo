# Skills Configuration for Luem-pai-laeo

## /screenshot
Description: Capture a screenshot of the running dev server to verify UI changes
Steps:
1. Use the browser or screen capture tool to take a screenshot of localhost dev server
2. Review the captured image for layout issues, alignment, spacing
3. Report findings and suggest fixes

## /verify-layout
Description: Verify that UI layout matches mobile dimensions and is user-friendly
Steps:
1. Check all screens render correctly at 360-428px width
2. Verify bottom nav doesn't overlap content
3. Verify modals are properly centered/positioned
4. Check touch targets are at least 44x44px
5. Verify text is readable (min 14px body, 12px captions)

## /check-thai
Description: Verify Thai language strings are properly displayed
Steps:
1. Check all UI strings have Thai translations
2. Verify Thai text renders correctly with proper font
3. Check text doesn't overflow containers
4. Verify date/time formats are localized

## /build-check
Description: Run build and verify no errors
Steps:
1. Run `pnpm run build`
2. Check for TypeScript errors
3. Check for missing imports
4. Verify build output size is reasonable

## /optimize-ui
Description: Run through all screens and optimize for mobile UX
Steps:
1. Check each screen: Home, Check, Trips, Find, Routines
2. Verify spacing, alignment, typography hierarchy
3. Check interactive elements have proper feedback (active states)
4. Verify scroll behavior and overflow handling
5. Check color contrast meets WCAG AA

## /generate-manual
Description: Generate user manual PDF with screenshots and descriptions
Steps:
1. Capture screenshot of each screen/flow
2. Write step-by-step descriptions for each use case
3. Organize by: Setup, UC1, UC2, UC3, UC4
4. Include Thai and English descriptions
5. Export as PDF with proper image placement

## /fix-bugs
Description: Scan all components for common layout and logic bugs
Steps:
1. Check for hardcoded widths that break responsiveness
2. Check for z-index conflicts
3. Check for missing key props in lists
4. Check for broken event handlers
5. Verify all buttons have proper onClick handlers
